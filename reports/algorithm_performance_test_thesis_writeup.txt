ALGORITHM PERFORMANCE EVALUATION OF MACHINE LEARNING CLASSIFIERS FOR STUDENT RISK PREDICTION IN AN ACADEMIC DECISION SUPPORT SYSTEM

1. INTRODUCTION
Algorithm performance testing is a critical phase in the development of decision support systems. It ensures that the selected model not only provides accurate predictions but also operates within the computational constraints of the deployment environment. In student risk prediction, actual measured values are necessary to prevent theoretical bias and ensure that the system can respond in real-time to student inquiries while maintaining high reliability in identifying at-risk individuals.

2. OBJECTIVE OF THE TEST
The objective is to evaluate the actual measured performance of five primary classifiers: Decision Tree, Random Forest, SVM, KNN, and Naive Bayes. The test specifically measures training execution time, prediction latency, peak memory usage, CPU utilization, and standard classification metrics (Accuracy, Precision, Recall, F1-Score) to determine the most suitable model for a production-grade academic monitoring system.

3. DEFINITION OF TECHNICAL TERMS
* Algorithm Performance Test: An empirical evaluation of an algorithm's speed, resource consumption, and output quality.
* Execution Time: The total duration required for an algorithm to complete a task.
* Training Time: The time taken to build the mathematical model from the training dataset.
* Prediction Latency: The time taken by a trained model to classify a new, single instance.
* Memory Usage: The amount of RAM consumed by the process during execution.
* CPU Utilization: The percentage of the central processing unit's capacity used by the task.
* F1-score: The harmonic mean of precision and recall, providing a balanced metric for imbalanced datasets.
* SMOTE: Synthetic Minority Over-sampling Technique, used to balance classes by generating synthetic examples of the minority class.
* Benchmarking: The process of comparing performance against a standard or between different candidates.

4. ALGORITHMS TESTED
a. Decision Tree: Uses a flowchart-like structure to reach conclusions based on feature thresholds.
b. Random Forest: An ensemble of decision trees that improves accuracy by averaging multiple predictions.
c. Support Vector Machine (SVM): Finds the optimal hyperplane that maximizes the margin between classes.
d. K-Nearest Neighbors (KNN): Classifies instances based on the majority label of their nearest neighbors in the feature space.
e. Naive Bayes: A probabilistic classifier based on Bayes' Theorem, assuming independence between features.

5. TEST DATA
* Filename: student_performance_dataset_v2.csv  
* Records: 101
* Features: 28 numeric features (Academic behaviors, GPA trends, LMS engagement, etc.)
* Target Variable: risk_label
* Class Labels: 0 (Good Standing), 1 (At-Risk)
* Preprocessing: Data cleaning, Mean Imputation for missing values, Standard Scaling, and SMOTE balancing on the training set.

6. PERFORMANCE METRICS
F1-Score was prioritized as the primary quality metric due to the inherent class imbalance in student risk levels. Computational metrics like Prediction Latency and Peak Memory Usage are used to assess the system's responsiveness and scalability.

7. TESTING PROCEDURE
The test was conducted using Python 3.x and the Scikit-learn library. Execution time was measured using 'time.perf_counter()' for microsecond precision. Memory and CPU metrics were captured via the 'psutil' library. A fixed random state of 42 was used for all splits and algorithms to ensure identical testing conditions. The dataset was split into 70% training and 30% testing sets using a stratified approach to preserve class distribution.

INTERPRETATION:
* Fastest Algorithm (Training): KNN (0.84 ms), though this is effectively model initialization.
* Fastest Algorithm (Prediction): Decision Tree (0.25 ms).
* Lowest Memory Usage: Naive Bayes (0.04 MB).
* Highest F1-Score: Random Forest (0.9700).
* Highest Computational Cost: Random Forest, requiring significantly more time and memory than other models.

9. COMPARATIVE ANALYSIS OF THE FIVE ALGORITHMS
There is a clear trade-off between predictive accuracy and computational resource consumption. Random Forest provides the highest reliability (97.0% F1) but consumes significantly more memory than Naive Bayes. However, even at its peak (26.4 MB), the resource usage is negligible for modern server environments. SVM emerges as a secondary choice, offering 89.1% accuracy with much faster training times than Random Forest.

10. INTERPRETATION OF RESULTS
The results indicate that the current dataset size (101 records) allows all five algorithms to perform within acceptable ranges for a web-based dashboard. The 7.8ms prediction latency of Random Forest ensures that the "Student Dashboard" will remain highly responsive even under concurrent user loads. As the dataset grows, the linear scaling of SVM and Naive Bayes may become more attractive, but Random Forest remains the current champion for prediction quality.

11. CONCLUSION
Based on the actual measured results, Random Forest is the best-performing algorithm for the Student Risk Prediction System. It achieved a superior F1-score of 0.9700, effectively distinguishing between different risk levels. While it is the most resource-intensive of the five, its performance is well within the limits of production readiness.

12. RECOMMENDATION
* Retain Random Forest as the primary prediction engine (Champion Model).
* Implement SVM as a fallback or shadow model for performance validation.
* Automate the logging of training time and memory usage during every model update.
* Re-run this benchmark once the student database exceeds 1,000 records to monitor scalability.
