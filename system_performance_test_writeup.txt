Title: 3.4 Performance Test

The performance test focuses on the backend system efficiency and computational resource utilization of the predictive models. This assessment is critical for ensuring that the Early Warning System can provide real-time feedback without exhausting server resources.

Key metrics monitored include Training Execution Time, Prediction Latency, Peak Memory Usage, and CPU Utilization. Naive Bayes and KNN demonstrated the lowest computational footprint, with Naive Bayes requiring only 1.24ms for training and 8.5% CPU utilization. 

Random Forest, while highly accurate, required the most resources with a training time of 115.32ms and 24.8MB of memory usage. Despite this, all models stayed within acceptable bounds for an educational management system, with prediction latencies remaining below 0.2ms per record.

Performance Summary:
- Fastest Training: Naive Bayes (1.24ms)
- Lowest Memory: Naive Bayes (16.8MB)
